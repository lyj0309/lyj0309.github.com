鑫火保险缴纳系统需求说明

本文件汇总鑫火商业保险缴纳系统（客户端 + 管理端）的业务流程、角色权限、功能需求与非功能要求，作为后续产品设计与研发实现的依据。

版本2025-11-01    编制产品需求整理

1. 背景与总体概述

鑫火商业保险缴纳系统服务于鑫火集团与旗下合作企业（约 300 家），目标是帮助企业录入员工信息、选择商保方案、完成缴费，并由鑫火集团业务员进行人工投保操作。系统分为两大端：

客户端（企业端）

  • 企业管理员和普通账号通过 Web/H5 访问。
  • 负责上传员工信息、配置投保方案、执行缴费、查看状态和账本。
  • 支持批量数据导入、方案默认值设定、余额抵扣等功能。

管理端（集团端）

  • 默认归属“鑫火集团”，无创建公司流程。
  • 按分组管理业务员及其所服务的企业，负责审核资料、人工投保、同步状态。
  • 配置投保方案、字段、账本管理、发票处理与全局统计分析。

系统强调分组管理：同一分组内的普通账号共享数据，管理员可纵览所有分组。分组与企业、业务员之间的绑定关系决定可见范围与操作权限。

2. 角色体系与权限模型

端 角色 主要权限 备注
客户端 企业管理员
  • 创建/管理企业、邀请或审批账号加入。
  • 创建分组、调整成员分组与权限。
  • 管理员工信息、投保方案、缴费、账本、发票。
  • 查看全量数据及站内信、待办事项。
 公司创建者默认为管理员且不可降级为普通账号；可配置多个管理员。
客户端 普通账户
  • 查看并维护所属分组的数据。
  • 录入、编辑本分组员工信息与方案。
  • 查看当月缴费、投保状态、历史记录。
  • 接收站内消息及待办提醒。
 权限来源于所属分组，切换分组后权限同步变更。
管理端 管理账号
  • 维护分组、配置业务员与企业绑定。
  • 全局配置（字段模板、投保方案、电子签章等）。
  • 查看全量日志、账本、统计报表，对企业执行跨分组可见设置。
 拥有系统最高权限，可访问所有分组和企业数据。
管理端 普通账号（业务员）
  • 维护所辖分组内企业的员工、方案、投保状态。
  • 执行人工缴费确认、退保处理、账本操作、发票开具。
  • 查看和维护本分组日志与备注。
 可手动将企业添加到本分组，可设置公司在多组间临时共享（含有效期）。

3. 账号与认证

3.1 登录与注册

  • 账号即手机号码，手机号唯一绑定单一账号，不支持更换手机号。
  • 支持两种登录方式：手机号 + 验证码登录；手机号 + 密码登录（需先在系统内启用并设置密码）。
  • 短信验证码无发送频率、错误次数或 IP 限制，验证码格式不限，按运营标准实现。
  • 密码策略：最少 8 位，连续 5 次登录失败后锁定账号 1 小时；无限制密码过期策略。
  • 首次注册流程：输入手机号，接收验证码后自动创建账号，随后可配置是否启用密码登录。

3.2 账户状态

  • 管理端可冻结账号登录（包括企业端与管理端账号）；冻结后数据保留，无法登录。
  • 暂不支持账号注销或手机号变更。

3.3 等待分配界面

  • 企业创建完成后立即生效，同时进入“等待分配专属业务员”界面。
  • 界面展示专属业务员姓名、手机、微信，并提供基于长连接的实时聊天功能（支持即时沟通）。
  • 可留言、发送即时消息（附件支持未明确，可后续补充）。

4. 企业、账号与分组管理

4.1 企业生命周期

  • 无所属公司的账号：可提交营业执照与法人身份证，立即创建公司并成为管理员。
  • 企业可由多名管理员共同管理，权限相同。创建者不可降级为普通账号。
  • 加入企业流程：管理员在客户端手动生成动态验证码（有效期 1 小时），普通账号凭公司名称 + 动态验证码加入，无需额外审批。

4.2 分组规则

  • 分组由客户端管理员与管理端业务员分别维护，决定普通账号的数据可见范围。
  • 普通账号分组变更后查看权限即时切换，无需站内信提醒或频率限制。
  • 企业或员工历史投保记录与操作日志保留原分组信息用于追溯。
  • 管理端可将某企业共享给其他分组，并设置共享有效期，期间具备同等操作权限。

4.3 业务员分配

  • 专属业务员由管理端管理员手动指定，无自动分配或 SLA 提醒。
  • 客户端待分配状态可通过待办提醒提示管理端进行处理。

5. 证照、资料与文档

5.1 证照上传

  • 支持上传营业执照、法人身份证、员工身份证等原版文件，无大小、分辨率限制。
  • 资料长期保存，支持再次上传；不支持版本回退，仅可查看历史记录。
  • 需记录每次上传的账号、时间及备注；备注受分组权限控制（管理员可见全部，普通账号仅见所属分组）。

5.2 备案文档与审批单

  • 支持生成备案文档与审批单，模板按当前页面信息直接生成。
  • 文档需包含流水号、电子签章、二维码校验；电子签章由管理端管理员配置。

5.3 批量导入

  • 提供 Excel 模板下载，企业填写后上传导入员工信息。
  • 导入失败整体回滚，不支持部分成功；错误提示以统一信息呈现（无需逐条原因）。
  • 导入日志需永久保存，便于事后审计。
  • 身份证识别与导入数据通过身份证号关联；若身份证号一致但姓名不同，拦截并提示“身份信息录入失败”。

6. 员工信息配置与核验

6.1 字段管理

  • 字段由管理端配置，可设置默认字段、为特定公司定制字段。
  • 配置项包含：必填/选填、字段类型（文本、数字、日期、枚举、多选等）、候选项列表、显示顺序、字段分组。
  • 配置修改后立即生效，无版本控制；客户端可选择隐藏或显示字段，但无法新增或删除字段。

6.2 身份证核验

  • 调用既定的第三方服务进行姓名 + 身份证一致性核验。
  • 核验失败直接驳回，提示“请上传清晰的身份证信息”。
  • 如需人工覆盖，可联系拥有该公司权限的管理端普通账号进行手工录入；需保留操作记录（人员、时间、原因）。
  • 无需记录第三方返回码或流水号。
  • 当前仅支持大陆身份证，不兼容其他证件。

7. 投保方案与计费规则

7.1 方案管理

  • 投保方案由管理端配置，可控制方案是否对特定企业展示。
  • 企业端管理员和普通账号（限分组）可为员工选择方案、批量设定默认值、单独调整。
  • 需确保“同一公司 + 同一方案 + 同一生效区间”内每名员工仅有一个有效条目，若需变更需先停用旧条目。

7.2 费用计算因子

费用实时计算，初版考虑以下维度（后续可扩展）：

  • 基础保费与计费周期（日/月/年）。
  • 工种风险系数、地区系数、年龄段系数。
  • 保障额度调整、附加险费用。
  • 活动折扣、优惠、余额抵扣。

7.3 生效期与叠加

  • 生效期可配置多段，跨月/跨年按实际天数折算。
  • 允许同一员工选择多条方案并存在生效期重叠；费用按实际数值叠加。
  • 前端需显示总保障金额与覆盖范围叠加提示，提醒潜在冲突。
  • 投保状态包括：待缴费、投保中、投保成功、投保失败等，可根据流程扩展。

7.4 退保与余额

  • 退保或提前终止将剩余金额计入企业余额，需记录原保单、原因、操作人、时间。
  • 支持部分退费，金额计算遵循实际剩余天数。

8. 支付、对账与账本

8.1 支付方式

  • 支持线上支付（支付宝、微信、银行卡等形式）与线下转账（无需上传凭证）。
  • 线上支付无需回传第三方交易流水；线下支付由企业自行转账后点击“确认支付”。
  • 确认后的核销由管理端业务员与财务人工处理。

8.2 应缴金额

  • 当月应缴金额实时计算，根据员工投保方案、生效时间、退保情况动态更新。
  • 若员工删除或方案调整导致未缴费用，应自动将相关金额退回企业余额。

8.3 企业账本

  • 账本需记录资金流水，区分余额类型（如退保余额、预存款等）。
  • 流水分类：充值、消费、退回、提现等，可导出明细报表。
  • 提现仅支持单一目标账户，由管理端服务人员审核并人工退款；无需系统生成或上传凭证。

9. 发票管理

  • 仅支持按总金额开具电子发票。
  • 发票申请需填写：发票抬头、税号、地址电话、开户行账号、发票类型、收件方式等字段，无需上传资质附件。
  • 发票金额需不超过已付款累计金额；多笔付款可合并开票。
  • 管理端完成开票后上传 PDF，可重新上传或作废重开；客户端可下载，无需确认收票。

10. 状态同步与通知

10.1 投保状态同步

  • 管理端提供批量设置投保状态界面，支持按公司、方案、分组筛选。
  • 不支持导入 CSV/Excel 更新，全部在线操作。
  • 每次状态变更自动写入操作日志，客户端管理员可查看结果。

10.2 消息通知

  • 站内信为唯一通知渠道，不支持短信、邮件、企业微信。
  • 无需分类、搜索或批量标记，消息永久保存；不支持提醒频率配置。
  • 客户端无需实时刷新机制，按用户访问时呈现最新数据。

11. 待办与提醒

  • 待办类型覆盖：未分配业务员、未完成支付、资料待补充、待审核提现、待开票等业务场景。
  • 根据角色展示不同待办列表（管理员、普通账号、管理端业务员）。
  • 暂不支持优先级、截止时间、转派；逾期不自动升级提醒。

12. 日志、审计与安全要求

12.1 日志策略

  • 保留所有类型的日志与操作记录，包括登录、查看、下载、数据增删改等。
  • 记录要素：操作人、时间、对象、详情（必要时包括字段级变更）、访问来源（IP、设备、浏览器等）。
  • 日志永久保存，支持导出；管理端普通账号查看本分组及其企业日志，管理账号查看全量。

12.2 风控策略

  • 无需对多地登录进行判断或预警；视为正常行为。
  • 暂不接入第三方风控系统。

13. 多端与移动适配

  • 提供响应式 H5 体验，确保移动端与桌面端功能一致。
  • 支持移动端相机上传身份证、实时聊天等核心功能的触控优化（按业务需求实现）。

14. 数据导出与统计

  • 管理端可导出企业或员工的投保历史、月度汇总、账本明细等报表。
  • 需要提供 API 日志或数据导出能力供内部 BI 使用，但系统不对客户开放外部 API。
  • 分组级别需显示当月统计、累计金额等指标，管理账号可查看全量统计。

15. 非功能性要求

  • 存储策略：所有业务数据、日志、资料永久保存，无数据留存期限限制。
  • 安全与合规：目前不要求额外加密、脱敏或特定安全认证，但需完整访问日志。
  • 性能假设：需支持 300 家企业并发使用，支持批量导入、实时费用计算场景。
  • 可维护性：操作日志完善，方便追溯；字段、方案配置须具备灵活扩展能力。

16. 后续可扩展项（参考）

  • 费用计算维度将根据后续业务补充进行扩展。
  • 实时聊天附件、消息推送等增强功能待进一步确认。
  • 若未来需要密码复杂度、验证码防刷、审批流程等高级安全策略，可在此基础上演进。
