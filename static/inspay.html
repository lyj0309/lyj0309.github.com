鑫火劳工对接平台需求说明

本文件梳理鑫火劳工对接平台（劳工端/企业端微信小程序 & 管理端 Web）的业务流程、角色权限、功能需求、风控策略和非功能要求，用于指导产品设计与研发实现。

版本2025-11-02    编制产品需求整理

1. 背景与总体概述

鑫火劳工对接平台面向劳工与企业的灵活用工场景，通过微信小程序实现劳工端与企业端按身份登录，并提供鑫火集团内部管理端 Web 系统用于运营、风控与合规。平台聚焦岗位匹配效率、身份核验、证件留存、任务执行追踪与资金结算安全。

劳工端/企业端（微信小程序）

  • 统一小程序入口，根据登录身份呈现差异化功能。
  • 覆盖岗位匹配、接单管理、任务打卡、评价互评、资金结算。
  • 接入微信生态能力（实名、定位、消息）以降低使用门槛。

管理端 Web

  • 鑫火集团内部员工使用，区分管理账户与普通账户。
  • 负责审核、风控、资金审批、配置管理、报表与日志审计。
  • 承载未来企业 PC 端扩展所需的接口与权限体系。

2. 角色体系与权限边界

角色 核心目标 主要权限 备注

3. 核心业务流程

3.1 注册与登录

  • 劳工/企业通过微信登录获取 code，后端换取 openid/unionid 并落库。
  • 首次登录需绑定手机号（短信验证），补充身份/企业信息与必需证件。
  • 管理端通过账号密码 + 二次验证登录（短信或企业微信）。

3.2 实名认证与资质审核

  • 劳工与法人实名认证接入微信实名系统，保留认证凭证。
  • 劳工特殊行业证件：上传 → 大模型识别 → 第三方权威库核验 → 更新状态与提醒。
  • 企业营业执照 OCR + 第三方企业风险信息校验（真伪、风险等级、历史记录）。

3.3 需求发布与匹配

  • 企业按照行业模板发布需求，支持公开抢单或定向邀约。
  • 劳工端支持按分类、位置、酬薪、技能、信誉筛选并提供个性化推荐。

3.4 接单执行与打卡

  • 工单确认后，劳工按到岗、阶段、完工节点进行打卡（微信定位 + 拍照水印）。
  • 无信号场景可申请补卡，须上传佐证资料并由企业或后台审核。

3.5 结算与互评

  • 企业选择即时结算或月度结算；平台自动扣取抽佣并生成对账单。
  • 劳工收入进入平台余额，依据阈值区分自动/人工审核提现。
  • 劳工与企业互评可追加内容，不允许删除；删除需后台管理端执行并留痕。

3.6 运营风控

  • 覆盖黑名单、异常打卡、证件到期、企业风险变更、争议处理等场景。
  • 所有审核与配置操作写入系统日志，支持追溯与导出。

4. 业务模式与资金流程

4.1 抽佣与结算

  • 平台抽佣模式，比例由管理端配置，可按行业、企业等级设置差异化策略。
  • 企业可选择工单即时支付或按月结算，系统需生成应付款清单及提醒。

4.2 余额与提现

  • 企业付款进入平台托管账户，扣除抽佣后计入劳工余额。
  • 支持微信零钱、银行卡、支付宝提现；多账户绑定需实名认证校验。
  • 小额提现：按日/月累计阈值内自动到账；阈值待确认。
  • 大额提现：超出阈值触发管理端审批（财务初审 → 财务主管复核 → 管理账户放款）。

4.3 商业保险代购

  • 特殊行业工单需自动代购商业保险，记录保单号、保额、保障期。
  • 保费由企业承担，可随工单结算；保险状态与任务执行打卡关联。
  • 后续需确认合作保险机构、费用结算与保单通知流程。

5. 功能需求详述

5.1 公共能力

  • 账号体系：微信登录、手机号绑定、验证码登录、账号合并、密码找回。
  • 消息中心：审核结果、打卡提醒、结算通知；基于微信订阅消息与站内信。
  • 即时沟通：平台自研 IM，支持一对一/群聊、图片/语音/位置；后台可查询。
  • 附件管理：统一文件服务，支持图片、PDF、视频上传，加密存储与权限控制。
  • 帮助与客服：FAQ、在线客服、意见反馈、争议申诉入口。

5.2 劳工端（微信小程序）

  • 个人中心：基础信息、技能标签、工作经历、可用时段、紧急联系人。
  • 证件库：身份证、健康证、特种证书、体检报告等；到期提醒与更新记录。
  • 岗位大厅：列表/地图模式、智能排序、收藏与分享、推荐位。
  • 接单流程：报名/抢单、企业反馈、确认上岗；特殊工种需完成证件验证。
  • 任务执行：工单列表、流程提醒、异常申报、补卡申请。
  • 收入中心：余额明细、佣金记录、提现申请、提现进度。
  • 评价系统：历史评价浏览、追加评价、对不实评价发起申诉。
  • 安全教育：特种作业须知、保险权益说明、线上培训资料。

5.3 企业端（微信小程序 & 未来 PC/Web）

  • 企业档案：基本信息、业务范围、联系人、证件状态、风险评分。
  • 角色管理：管理员、招聘专员、财务、现场负责人，支持细粒度权限。
  • 需求发布：行业模板、批量导入、复制历史、草稿/发布/暂停/关闭管理。
  • 候选管理：报名列表、简历预览、标签筛选、邀约与沟通、进度记录。
  • 现场管理：指派负责人、查看打卡轨迹、异常提醒、补卡审批。
  • 结算中心：工时确认、抽佣核算、账单导出、发票申请。
  • 评价与风控：劳工评价、标签、黑名单同步；风险预警提示。
  • 合同文档：合同模板上传、下载，后续支持电子签章对接。

5.4 管理端 Web

  • 权限中心：角色定义、菜单/按钮权限、审批流配置、二次验证策略。
  • 用户与证件审核：实名审批、特殊证件复核、证件到期提醒、黑名单维护。
  • 资金管理：抽佣配置、企业账期、余额监控、大额提现审批、资金流水。
  • 风控中心：风险规则、企业风险报告接入、争议处理、事件跟踪。
  • 配置中心：岗位分类、证件类型、消息模板、地理围栏、广告位等。
  • 日志中心：操作日志、访问日志、审批记录检索与导出。
  • 报表中心：多维度统计仪表盘，支持钻取与导出。

6. 非功能性要求

  • 安全：全链路 HTTPS；敏感数据静态加密；二次验证；防刷、限频、防注入；聊天内容审计。
  • 隐私与合规：遵循《个人信息保护法》《数据安全法》；用户数据可导出/注销；证件访问留痕。
  • 性能：小程序接口响应 < 3 秒；支持 ≥10 万注册劳工、日活 2 万；打卡提交高并发防护。
  • 可用性：核心服务 99.9% SLA；日志与证件多副本存储。
  • 扩展性：服务组件化，便于未来接入 PC/H5/APP。
  • 运维：监控告警、巡检报表、运维操作纳入审计。
  • 体验：适配主流机型。

7. 集成与技术考虑

  • 微信生态：code2Session、订阅消息、定位、拍照、文件上传、地图组件。
  • 实名服务：微信实名认证；保留认证凭证。
  • 证件识别：大模型 OCR/分类（自研或第三方），失败时提供人工复核入口。
  • 企业风险：接入第三方企业风险信息服务，返回风险等级与事件标签。
  • 地图服务：腾讯位置服务，用于打卡定位与岗位地图展示。
  • 消息与通信：自研 IM + 微信订阅消息，后续可扩展短信或企业微信通知。
  • 存储方案：对象存储（OSS）+ 数据库元数据，开启加密、生命周期、访问控制。
  • 日志检索：ElasticSearch + Kibana 或同类方案，支撑日志查询与风控分析。
  • 支付链路：支持银行转账、微信零钱、支付宝；大额提现结合审批后执行。

8. 数据与存储策略

  • 核心数据模型：用户主数据、证件、技能标签、岗位、工单、打卡、评价、聊天记录、资金流水、操作日志。
  • 证件与打卡照片长期保存 ≥5 年或依政策调整，支持冷存储与冗余备份。
  • 数据脱敏：导出报表对身份证号、手机号实现掩码；后台查看需写入访问日志。
  • 日志保留：操作日志 ≥2 年，系统日志 ≥6 个月；支持合规留存要求。
  • 数据访问审批：企业查询劳工证件需记录审批链路与访问目的，避免滥用。

9. 运营与增长支持

  • 拉新工具：邀请奖励、二维码分享；需配套风控策略防刷单。
  • 活动配置：限时补贴、岗位置顶、推荐位管理，可按行业/地区配置。
  • CRM 触达：与企业微信、短信、外呼系统对接（后续扩展）。
  • 数据驱动：A/B 测试、留存与转化漏斗、用户画像分析。

10. 管理端审批流程与操作链路

10.1 账户审批

  • 管理账户开通：提交人员信息 → 系统校验员工编号 → 超级管理员审批 → 分配角色 → 首次登录绑定二次验证。
  • 普通账户开通：管理账户发起 → 部门负责人审批 → 系统通知 → 可按敏感度追加财务/风控审批。

10.2 业务审批示例

  • 大额提现：普通账户初审 → 财务专员审核 → 财务主管复核 → 管理账户放款并归档。
  • 抽佣比例调整：运营提出 → 运营总监审批 → 财务确认 → 管理账户发布生效。
  • 证件复审/黑名单解除：客服提交 → 风控主管审批 → 运营负责人确认执行。
  • 配置发布：普通账户编辑 → 生成变更单 → 管理账户审核后上线。

10.3 审批系统要求

  • 审批单状态：草稿、待审批、已通过、已驳回、已撤销。
  • 记录审批意见、附件、逾期提醒，支持导出存档。
  • 审批操作同步写入日志，供审计和风控追溯。

11. 数据报表与可视化

  • 日报/周报：注册实名、需求发布、成交工单、流水、提现、异常事件等核心指标自动生成。
  • 可视化：仪表盘、趋势图、热力图、漏斗图，支持按时间、地区、行业、企业类型筛选。
  • 推送：站内订阅、邮件、企业微信通知，可配置订阅人群与频率。
  • 导出格式：PDF、Excel；支持报表任务调度与历史查询。

12. 合规与备案要求

  • 需完成 ICP 备案、公安网安备案，并在管理端存档备案信息与年审提醒。
  • 证件长期保存需基于劳工注册时签署的授权协议，协议内容留存可追溯。
  • 资金合规：评估第三方资金托管或银行监管账户需求，满足监管要求。
  • 聊天组件需遵守内容安全要求，提供敏感词过滤与记录留存能力。

13. 扩展规划

  • 企业 PC/Web 端：规划扫码登录/账号密码登录、批量管理、报表导出等桌面特性；与小程序共享权限与数据。
  • 劳工 H5 端暂未启动，但接口与权限设计需支持未来多端扩展。
  • 暂不考虑境外或多语言，若未来扩张需重新评估合规、风控与语言策略。

14. 待确认事项

  1. 小额/大额提现阈值建议范围（如日累计 ¥5,000、月累计 ¥20,000），确认后固化风控规则。
  2. 商业保险合作机构、投保模式（按单/按月）、保费结算方式与保单通知渠道。
  3. 大模型识别服务的选型、成本、准确率要求，以及识别失败的回退流程。
  4. 企业风险信息供应商名单与 SLA 标准。
  5. 聊天记录留存期限、查询权限、敏感词审查规范。
  6. 企业 PC/Web 端上线节奏与阶段划分，是否与小程序同步发布。
  7. 评价追加次数或时间窗口限制方案，防止滥用。
  8. 资金托管方案与监管要求，是否需要第三方托管账户。
